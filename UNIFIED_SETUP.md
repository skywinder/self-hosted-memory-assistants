# Unified Setup

This repository now has a single entry point for bootstrapping all three subprojects from the meta-repo root.

## Quick Start

```bash
cp .setup.env.example .setup.env
./setup.sh
```

If you also want to start the configured services:

```bash
./setup.sh --start
```

To preview what would happen without changing files:

```bash
./setup.sh --dry-run
```

To configure only part of the workspace:

```bash
./setup.sh --projects mycelia,ushadow
```

## What `./setup.sh` Does

- Uses one shared config file: `.setup.env`
- Creates missing runtime config for `mycelia`, `chronicle`, and `ushadow`
- Generates local secrets where the projects expect them
- Preserves existing generated secrets if they already exist
- Optionally starts the configured projects

## Notes Per Project

### Mycelia

- `./setup.sh` prepares `mycelia/.env`
- Mycelia's own CLI tokens `MYCELIA_CLIENT_ID` and `MYCELIA_TOKEN` are still generated by Mycelia itself, either through its UI setup flow or `./scripts/setup.sh --with-tokens`

### Chronicle

- The unified setup creates `config/config.yml`, `config/plugins.yml`, root env files, and `backends/advanced/.env`
- Speaker recognition env is created only when `HF_TOKEN` is present in `.setup.env`

### Ushadow

- The unified setup writes `ushadow/.env`
- It also creates or updates `ushadow/config/SECRETS/secrets.yaml`
- `ushadow` keeps API keys in `secrets.yaml`, while ports and compose settings live in `.env`
